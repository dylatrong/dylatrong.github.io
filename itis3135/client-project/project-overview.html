Client Project: Requirements & Design Proposal

1. Project Overview

Project Name:

3communiTEA at UNCC

Application Description & Purpose:

The "3communiTEA at UNCC" website will serve as the central digital hub for the UNC Charlotte tea club. Its primary purpose is to enhance the club's visibility, streamline communication, and foster a stronger community. The application will provide essential information to both prospective and current members, including event schedules, club mission, and membership details. It aims to improve recruitment by offering a clear and engaging entry point for interested students and to keep current members informed about activities.

Intended Users:

  • Prospective Members: Current UNCC students interested in joining a new organization.
  • Current Members: Existing club members looking for event information, updates, and resources.
  • Club Officers (E-board): Users who will manage and provide the content (though they will not be ableto update it directly via the site).
  • General UNCC Community: Faculty, staff, or other student groups looking to collaborate.

Content Overview:

  • About Us: Information on the club's mission, values, and the current e-board members.
  • Events: An interactive calendar and list of upcoming meetings, special events, and collaborations.
  • Gallery: Photos and videos from past club activities and events.
  • Join Us / Contact: A simple, clear form for interested students to get on the mailing list and details on how to officially join, alongside social media links.

5. Page Design

Page 1: Home (index.html)

  • Purpose: To welcome users, provide a brief overview of the club, and direct them to key areas (events, joining).
  • Audience: All users (Prospective, Current, General).
  • Content: Hero image, a short "Welcome to 3communiTEA" blurb, a call-to-action button to "Join Us," and a small preview of the next upcoming event.
  • Data Entry: None.
  • Validations: N/A.
  • Buttons/Links: Main navigation bar, "Join Us" button (links to join.html), "See More Events" link (links to events.html).
  • Actions: Navigation to other pages.
  • Notes: This page must be visually engaging to capture interest immediately.

Page 2: About Us (about.html)

  • Purpose: To detail the club's mission, history, and introduce the leadership team.
  • Audience: Prospective members, general UNCC community.
  • Content: "Our Mission" section, and "Meet the E-Board" section with photos and brief bios for each officer.
  • Data Entry: None.
  • Validations: N/A.
  • Buttons/Links: Main navigation. May include social media icons.
  • Actions: None, information-only.
  • Notes: Bios and photos will be provided by the client.

Page 3: Events (events.html)

  • Purpose: To inform all users about upcoming and past club meetings and special events.
  • Audience: All users, especially current members.
  • Content: A jQuery-UI Calendar widget and a chronological list of events. Each event will have a title, date, time, location, and short description.
  • Data Entry: None (on the front-end).
  • Validations: N/A.
  • Buttons/Links: The calendar dates will be interactive.
  • Actions: Clicking a date on the calendar will use AJAX to fetch event details from a local events.json file and display them on the page without a full reload.
  • Notes: This is a key page for member engagement. The JSON file will act as our read-only database.

Page 4: Gallery (gallery.html)

  • Purpose: To showcase the club's activities and community atmosphere through photos.
  • Audience: Prospective members, current members.
  • Content: A grid of thumbnail images from past events.
  • Data Entry: None.
  • Validations: N/A.
  • Buttons/Links: Each thumbnail will be a link.
  • Actions: Clicking a thumbnail will open the full-size image in an interactive "lightbox" (jQuery plugin), allowing users to cycle through the gallery.
  • Notes: This will use a third-party jQuery plugin like Lightbox2.

Page 5: Join Us / Contact (join.html)

  • Purpose: To provide a clear path for new members to join and a way for anyone to contact the club.
  • Audience: Prospective members, general UNCC community.
  • Content: A "Membership Interest" form, links to the club's social media (e.g., Instagram, Discord), and the club's official email address.
  • Data Entry: Yes, a form with fields for: Name, UNCC Email, and a checkbox for "Add me to the mailing list."
  • Validations: Yes. The "Name" field will be required. The "UNCC Email" field will be required and must use type="email" for browser-side validation.
  • Buttons/Links: A "Submit" button for the form. Social media icons/links.
  • Actions: The form's "Submit" button will trigger an onsubmit JavaScript event that validates the inputs and then displays a simple "Thank you for your interest!" message on the page. (Note: No data will be stored per course constraints).
  • Notes: This page is the primary "call to action" for recruitment.

Dynamic Functionality Page(s) Purpose & Description Example URL(s)
jQuery-UI Widget (Calendar) events.html To provide a familiar, interactive calendar interface. Users can visually see which dates have events. We will use the jQuery-UI Datepicker configured as an inline calendar. jqueryui.com/datepicker
AJAX Functionality (Event Loading) events.html When a user clicks a date on the calendar, an AJAX (Fetch API) request will be made to a local events.json file. JavaScript will parse this file, find the event corresponding to the selected date, and dynamically inject the event details (title, time, description) into a div on the page without requiring a page reload. MDN Fetch API Docs
jQuery Plugin (Lightbox Gallery) gallery.html To provide a professional, user-friendly way to view a gallery of images. Clicking a thumbnail will overlay the full-size image, and users can navigate between images with on-screen arrows. lokeshdhakar.com/lightbox2
JavaScript Form Validation & Interaction join.html Beyond standard HTML5 required attributes, JavaScript will be used on form submission to double-check the email field (e.g., ensuring it contains "@uncc.edu"). On successful validation, JS will hide the form and show a "Success!" message.
